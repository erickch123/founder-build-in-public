# Decisions and Roadmap

## Confirmed product decisions

| Topic | Decision |
|---|---|
| Repository | `founder-build-in-public` |
| Target | Technical solo founders and indie hackers |
| Surface | Terminal-first |
| Framework | Hermes skill around a standalone CLI |
| Core inputs | Curated Gmail newsletters + manual captures |
| Optional inputs | Git, Notion, calendar |
| Canonical store | Local structured files |
| End output 1 | Daily or weekly Founder Digest |
| End output 2 | Faceless 45–60 second vertical video |
| Video engine | Remotion |
| LLM | Low-cost OpenAI API model, configurable |
| Development assistant | Codex through ChatGPT Pro |
| Storage | Existing private GCS bucket |
| Publishing | Not included; review-ready draft only |
| Demo | Submission video generated by the product |
| Privacy | Workspace isolation and public-safe manifest |
| Hackathon goal | Low-effort, high-impact Top-30-quality submission |

## Product-language decisions

Use:

- Founder Digest
- Founder Reel
- workspace
- public-safe manifest
- curated evidence
- story compiler

Avoid leading with:

- generic AI video generator;
- life surveillance;
- fully autonomous operating system;
- vague multi-agent claims;
- replacement of an entire marketing team.

## MVP versus future

### MVP

```text
Gmail or fixture candidates
→ human selection
→ structured learning
→ local founder log
→ manual progress captures
→ workspace/privacy filter
→ story plan
→ digest
→ Remotion reel
→ private GCS link
```

### Near-term

- Notion export;
- Git import;
- weekly scheduled job;
- multiple visual themes;
- better action scheduling;
- project progress over multiple days.

### Longer-term

- calendar plan-versus-reality;
- multi-user profiles;
- self-hosted deployment;
- approval-based publishing;
- richer second-brain integrations;
- agent-generated content calendar;
- metrics and learning feedback;
- storage provider abstraction for S3.

## Multi-context model

A full-time workspace may be closed independently:

```bash
founder erick cadt close
```

The overall day remains personal:

```bash
founder erick default end-day
```

The overall story may combine only allowed summaries across workspaces.

## Future native Hermes UX

Desired experience:

```bash
hermes founder erick default end-day
```

MVP-compatible experience:

```bash
hermes -s founder-build-in-public -q \
  "Run end-day for erick in default"
```

A native positional command can be explored later through:

- a Hermes plugin;
- a wrapper executable;
- a shell function;
- an upstream extension point.

Do not let this syntactic preference delay the core artifact pipeline.

## Strategic moat

The long-term differentiator is not rendering. It is:

- founder-specific curation;
- workspace-aware memory;
- privacy boundaries;
- longitudinal story reconstruction;
- reusable evidence-to-media compilation.

The video renderer can change. The structured founder memory and editorial reasoning are the durable product layer.
