# Hackathon Checklist

## Before implementation

- [ ] Confirm repository is `founder-build-in-public`.
- [ ] Confirm all submitted code is fresh.
- [ ] Add the participant handbook summary to planning docs.
- [ ] Decide solo or two-person team.
- [ ] Confirm laptop, charger, and credentials are available.
- [ ] Prepare only allowed planning materials before the hacking period.

## Repository

- [ ] Public GitHub repository.
- [ ] Descriptive README.
- [ ] MIT or chosen license.
- [ ] `.gitignore`.
- [ ] `.env.example`.
- [ ] No credentials or private data.
- [ ] Frequent, understandable commits.
- [ ] Third-party credits.
- [ ] Architecture diagram.
- [ ] Demo command.
- [ ] Known limitations.
- [ ] Fixture data is clearly synthetic or sanitized.

## Fresh Code Rule

- [ ] Repository and implementation began during the official hacking period.
- [ ] No prior private project imported.
- [ ] No prebuilt custom component copied.
- [ ] Open-source dependencies are credited.
- [ ] Commit history demonstrates progression.
- [ ] Any AI coding assistance is acceptable and honestly represented.

## Golden path

- [ ] `capture` works.
- [ ] fixture inbox works.
- [ ] curated selection works.
- [ ] structured learning extraction works.
- [ ] privacy filter works.
- [ ] digest is generated.
- [ ] Remotion MP4 renders.
- [ ] video is 9:16.
- [ ] captions are readable.
- [ ] cloud upload works or local fallback is ready.
- [ ] self-generated submission video exists.

## Privacy

- [ ] Gmail scope is read-only.
- [ ] Only selected emails are processed.
- [ ] Real email bodies are not committed.
- [ ] Confidential workspaces are denied from public output.
- [ ] Renderer consumes only public-safe manifest.
- [ ] GCS bucket/object is private.
- [ ] Signed URL expires.
- [ ] No copyrighted football footage.
- [ ] No private person’s data appears without permission.

## Demo

- [ ] One-sentence pitch memorized.
- [ ] Live command tested from a clean terminal.
- [ ] Fixture mode does not require OAuth.
- [ ] Local MP4 plays without internet.
- [ ] Backup screen recording exists.
- [ ] Digest opens locally.
- [ ] Terminal output is concise.
- [ ] Generated video contains the self-reference reveal.
- [ ] Demo stays under expected pitch time.

## Submission

- [ ] OpenArena entry found.
- [ ] Project name and description complete.
- [ ] Team information complete.
- [ ] Repository URL correct and public.
- [ ] Credits complete.
- [ ] Submit before 6:00 PM SGT.
- [ ] Verify submission page after saving.
- [ ] Remain physically present for Top 30, Top 10, judging, and closing.

## Prize reality

- [ ] Understand that Top-30 AWS credits remain subject to company and AWS eligibility.
- [ ] Do not add AWS solely for the prize.
- [ ] Use the existing GCP bucket for implementation reliability.
- [ ] Treat Claude membership and AWS credit redemption as separate from product architecture.

## Final five-minute check

```text
git status
git log --oneline
secret scan
fixture end-to-end run
open digest
play MP4
verify repository visibility
verify OpenArena submission
```
